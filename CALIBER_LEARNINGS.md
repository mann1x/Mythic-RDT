# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[gotcha:project]** `scripts/_dscoder_compat.py::load_dscoder_tokenizer()` does `sys.path.insert(Path(base_path).resolve())` then imports `tokenization_deepseek_fast`. Passing an HF id (`deepseek-ai/DeepSeek-Coder-V2-Lite-Instruct`) as `--student` resolves to a non-existent local dir → `ModuleNotFoundError: No module named 'tokenization_deepseek_fast'`. Always pass a LOCAL path that contains the file. Pre-flight every pod chain with a 30 s tokenizer-load smoke test BEFORE the precompute/training compute.
- **[gotcha:project]** Mythic-RDT's `inject_depth_lora` resolves dotted paths only — passing bare leaf names like `up_proj,gate_proj,down_proj` against DSC-V2-Lite's MoE layers (FFN at `mlp.shared_experts.up_proj` and `mlp.experts[0..63].up_proj`) silently injects 0 LoRA modules. Use a recursive `named_modules` walk to expand bare leaf names (PEFT-style substring match), or pass full dotted paths. Verify post-inject with `LoRA modules wired: <N>` log line — if 0, the injector silently no-op'd.
- **[gotcha:project]** `torch.utils.checkpoint.checkpoint(..., use_reentrant=False)` truncates the backward graph if NO input tensor to the checkpointed function requires grad. With NF4-frozen base + frozen prelude, `h` arriving at the recurrence loop has `requires_grad=False` even though LoRA modules INSIDE require grad → no gradient flows. Fix: `if use_ckpt and not h.requires_grad: h.requires_grad_(True)` immediately before the loop.
- **[gotcha:project]** `self.recurrence.to(dtype=base_dtype)` without `device=` leaves LTI's `A_diag`/`B`, LayerScale, and gate parameters on CPU — but the wrapper crashes only at the FIRST iteration that exercises LTI (t=1 with `first_iter_identity`). With curriculum starting at T=1 and ramping at step 15-30, the device crash hides for many steps. Always move recurrence cell with both: `self.recurrence.to(dtype=base_dtype, device=base_device)`.
- **[gotcha:project]** SQLite `sqlite3 db.db "SELECT ... WHERE task=\"humaneval\""` over `ssh "..."` double-escapes and parses `humaneval` as a column name. Fix: single quotes inside the SQL (`task='humaneval'`) AND wrap outer ssh in double-quotes, OR pipe a heredoc SQL file with `sqlite3 db.db < /tmp/q.sql`.
- **[gotcha:project]** Caliber stop-hook `caliber-check-sync.sh` blocks every Claude Code `Stop` event in a git repo without `caliber` in `.git/hooks/pre-commit`. Run `/setup-caliber` once per repo to install the hook.
- **[gotcha:project]** Eval JSON result records (`eval_results/v6X_ckptN_lcb10_local.json`) do NOT contain raw generation strings. Per-result schema is `{task_id, passed, reason, cleaned_head, gen_secs, score_secs}`. Top-level `summary` block holds `pass_at_1`/`n_pass`/`n_total`. Read `cleaned_head` (max ~160 chars) and `reason` for spot-checks.
- **[gotcha:project]** `humaneval_smoke.py` writes `output_json` ONCE at end-of-run per T-value (atomic JSON dump, no incremental writes); intermediate state is NOT in any sqlite. Spot-checking pass@1 mid-run requires counting log lines (`grep -cE 'smoke wT=N HE.*PASS'` vs total) — the JSON only updates between T-values.
- **[pattern:project]** Spot-check pass@1 by grepping the run log: `grep -cE "smoke wT=$T HE" $LOG` for total + `... HE.*PASS` for passes. Useful for live progress when no JSON is flushed yet. Pair with `grep -E 'PHASE|exit_code' $LOG` to confirm phase boundaries.
- **[pattern:project]** Pre-quantized teacher caches live at `/workspace/cache_nf4/<model>-nf4/` on the pod. Their `config.json` carries an embedded `quantization_config` (BitsAndBytesConfig nf4 + double_quant). Load with `--quant none` and pass the dir as both `--teacher` and `--teacher-tokenizer`.
- **[pattern:project]** Loss-line tail extraction from wandb-logged training: use `tail -40 $LOG | grep -oE '(ce_gap|kl_anchor|margin_loss).{0,18}'`. Avoid `grep -oE "'(key)':\s*[-0-9.e]+"` — quote-escape inside `ssh "..."` breaks the regex.
- **[pattern:project]** When deferring a queued pod-side run from solidPC without killing the in-flight job, replace the queued runner script with a TRAMPOLINE that `exec`s into the new chain: `exec bash /workspace/.../run_vXX_chain.sh`.
- **[pattern:project]** Every joint/chain training script MUST run a tokenizer + base + wrapper + 1-microbatch-with-T>1 forward smoke test as the FIRST 30 s, BEFORE any precompute or curriculum-warmup compute. Gate the chain on a `--smoke-test` flag that exercises T>1 explicitly.
- **[pattern:project]** Long pod-side rsync: launch with `nohup rsync ... > /tmp/log 2>&1 &` and watch via `until ! kill -0 $PID; do sleep 60; done` in a `Bash run_in_background:true` task — survives Claude turn boundaries. `--info=progress2` writes `\r`-overwritten lines, so for tailing use `tr '\r' '\n' | tail -3`.
- **[pattern:project]** wandb log keys are namespaced `train/ce_loss`, `train/distill_loss`, `train/T_sample`, `train/lr`, `train/mask_density`, `train/epoch` — NOT bare `ce`/`distill`/`T`. `scan_history()` without explicit `keys=` returns rows where unrequested keys come back as `None`; pass `keys=[...]` to get all metrics from the same step.
- **[convention:project]** CTD pod artefacts mirror to `/srv/.../backup_models/ctd-pod-archive/<student-family>/{adapters,runs,results,logs}/`. Mirror order before destroying any pod: results JSONs → logs → runs (`rsync --partial --info=progress2`) → teacher caches. NEVER delete pod-side `runs/` until `du -sh` size parity passes. Full procedure in CTD `docs/POD_ARCHIVE.md`.
- **[gotcha:project]** `humaneval_smoke.py`'s `_run_base_eval_incremental` resume-from-JSON path expects each prior result to carry a `"passed"` key, but per-result records use `{task_id, passed, reason, cleaned_head, ...}` schema only when written by the modern path — older/partial JSONs raise `KeyError: 'passed'` at `state.results[...][r['passed']]`. When an HE-164 run crashed mid-eval and you re-launch with the same `--output-json`, the cleanest recovery is to delete the stale JSON entirely (`rm v6X_he164.json`) — partial-T deletion via JSON surgery is brittle.
- **[gotcha:project]** `humaneval_smoke.py`'s smoke loader injects DepthLoRA from CLI defaults (`self_attn.q_proj_or_q_a, self_attn.o_proj` rank 8) and then calls `_load_trainable_state` — when the ckpt was trained with a different LoRA tree (e.g. FFN-on-experts rank 24, 7414 keys), the load silently no-ops with `loaded 4 trainable tensors  missing=76 unexpected=7410` and the wrapper runs at INIT, making T=1=T=2=T=4=BASE. ALWAYS verify post-load report shows `missing=0 unexpected=0`. The smoke now autodetects targets+rank from the saved state_dict by default; pass `--no-lora-autodetect` only for intentional partial-load ablations.
- **[pattern:project]** When an eval script crashes mid-multi-phase (HE Phase 2a fails but LCB Phase 2b is still running) and you want to re-run just the failed phase WITHOUT killing the in-flight one: write a waiter script (`while pgrep -f 'humaneval_smoke.py.*<phase_marker>'; do sleep 30; done`) and `nohup` it in the background. It auto-launches the rerun the moment the GPU frees, with no manual reattach.
- **[gotcha:project]** In `recurrence.py`'s block_mode_residual branch, the LTI `injection` was historically computed but DROPPED from `h_next` (only `h + mix * (block_out - h)` was used), making `LTIInjection.A_diag` and `B_proj` dead weight (no forward use → no gradient → no learning). Any v6 variant relying on LTI signal must verify `lti_residual_scale > 0.0` is plumbed through `MythicRDTDeepseekV2Config` → `RecurrenceCell` → forward, otherwise LTI is silently inert.
- **[pattern:project]** When adding a new RecurrenceCell hyperparameter, plumb it through ALL FOUR sites or it silently defaults: (1) `configuration.py` MythicRDTDeepseekV2Config kwarg + self-assign, (2) `modeling.py` RecurrenceCell(...) construction with `getattr(config, ..., default)`, (3) `recurrence.py` RecurrenceCell `__init__` kwarg + self-assign + forward use, (4) CLI in `scripts/06_train_sft_recurrence.py` + `scripts/finetune_phase1.py` + `scripts/humaneval_smoke.py`. Missing the smoke script means eval re-loads ckpts at the default value, masking a real training effect.

Catches plumbing bugs in <10 s before a 10 h pod run.
