# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) â€” do not edit manually.

- **[correction:project]** When pivoting a recurrence training variant to use a teacher/recipe family already validated by the cross-tokenizer-distill (CTD) sister project, do NOT default to the generic v6R recipe â€” port the exact M-series recipe (M37c, M42q, M124f-style) that won on the SAME base+benchmark in CTD. The CTD `baselines/dscv2lite/` JSONs and `docs/PLAN_DSCV2LITE_M37C.md` are the source of truth for what works on DSC-V2-Lite-Instruct; recurrence is the additive on top.
- **[gotcha:project]** Vast.ai pod (200 GB overlay) fills fast. Before launching a precompute that needs another teacher download + cache shard writes, check `df -h /workspace` and `du -sh /workspace/cross-tokenizer-distill/experiments/validation/runs/*` â€” the runs/ dir is the canonical thing to mirror+prune.
- **[pattern:project]** Pre-quantized teacher caches live at `/workspace/cache_nf4/<model>-nf4/` on the pod. Their `config.json` carries an embedded `quantization_config` (BitsAndBytesConfig nf4 + double_quant). Load them with `--quant none` and pass the dir as both `--teacher` and `--teacher-tokenizer` â€” transformers auto-applies the embedded quant and saves a 28 GB redownload + re-quant.
- **[convention:project]** CTD pod artefacts are mirrored to `/srv/.../backup_models/ctd-pod-archive/<student-family>/{adapters,runs,results,logs}/`. Mirror order before destroying any pod: results JSONs â†’ logs â†’ runs (large, use `rsync --partial --info=progress2`) â†’ teacher caches. NEVER delete pod-side `runs/` until `du -sh` shows size parity with the local mirror. Full procedure in `docs/POD_ARCHIVE.md` (CTD repo).
- **[gotcha:project]** Quoting Python here-doc through `ssh "<<PYEOF ... PYEOF"` chokes on inline shell metacharacters (`2>&1`, `||`, multi-statement `import` chains compressed into one line). Use the single-quoted `ssh '...python <<PYEOF ... PYEOF'` form and write the python as multi-line code.
- **[gotcha:project]** `scripts/_dscoder_compat.py::load_dscoder_tokenizer()` does `sys.path.insert(Path(base_path).resolve())` then imports `tokenization_deepseek_fast`. Passing an HF id (`deepseek-ai/DeepSeek-Coder-V2-Lite-Instruct`) as `--student` resolves to a non-existent local dir â†’ `ModuleNotFoundError: No module named 'tokenization_deepseek_fast'`. Always pass a LOCAL path (e.g. `base/DeepSeek-Coder-V2-Lite-Instruct-nf4`) that contains the file. Pre-flight every pod chain with a 30 s tokenizer-load smoke test BEFORE the precompute/training compute.
- **[gotcha:project]** Mythic-RDT's `inject_depth_lora` resolves dotted paths only â€” passing bare leaf names like `up_proj,gate_proj,down_proj` against DSC-V2-Lite's MoE layers (where FFN lives at `mlp.shared_experts.up_proj` and `mlp.experts[0..63].up_proj`) silently injects 0 LoRA modules and `[lora] skip layer N up_proj: 'DeepseekV2DecoderLayer' object has no attribute 'up_proj'` is logged. Use a recursive named_modules walk to expand bare leaf names (PEFT-style substring match), or pass full dotted paths. Verify post-inject with `LoRA modules wired: <N>` log line â€” if 0 the injector silently no-op'd.
- **[gotcha:project]** Mythic-RDT's `MythicRDTDeepseekV2ForCausalLM.forward` returns a raw `torch.Tensor` (logits) by default and a `CausalLMOutputWithPast` only when `return_dict=True` OR `labels is not None` OR `use_cache=True`. Custom training loops must pass `return_dict=True` and unwrap with `s_out.logits if hasattr(s_out, 'logits') else s_out` â€” otherwise `s_out.logits` raises `AttributeError: 'Tensor' object has no attribute 'logits'`.
- **[gotcha:project]** Cross-vocab teacher cache `alignment_mask` raw mean is misleadingly low (~6%) because precompute zeros prompt-side positions. Compute completion-only coverage by masking with `(indices.abs().sum(-1) > 0)` (non-prompt positions) before averaging â€” proper coverage on funcsig corpus is ~100% over completion span.
- **[gotcha:project]** SQLite `sqlite3 db.db "SELECT ... WHERE task=\"humaneval\""` over `ssh "..."` double-escapes and parses `humaneval` as a column name. Fix: use single quotes inside the SQL (`task='humaneval'`) AND wrap the outer ssh in double-quotes, OR write the SQL to a heredoc file and pipe `sqlite3 db.db < /tmp/q.sql`.
- **[gotcha:project]** Eval `--chat-template` flag wraps prompts via `tokenizer.apply_chat_template` with `add_generation_prompt=True`. DSC-V2-Lite under chat template emits prose preambles that the raw `exec(prompt+gen)` scorer cannot execute, dropping HE pass@1 from 75.6% (raw) â†’ 10.4% (chat). For BASE re-eval baselines that need apples-to-apples with raw-completion training recipes, do NOT pass `--chat-template`.
- **[gotcha:project]** Caliber stop-hook `caliber-check-sync.sh` blocks every Claude Code `Stop` event in a git repo without `caliber` in `.git/hooks/pre-commit`. Run `/setup-caliber` once per repo to install the hook, or accept the nag.
- **[gotcha:project]** Eval JSON result records (`eval_results/v6X_ckptN_lcb10_local.json`) do NOT contain the actual generation strings under `generations` or `samples`. Per-result schema is `{name, pass_at_1, n_pass, n_total, elapsed_sec, failures, per_problem}`. Read `result['failures'][i]['completion_head']` and `result['failures'][i]['reason']`.
- **[correction:project]** Once a v6 variant shows the v6A failure pattern at ckpt-100 LCB-10 (wrapper 0/0/0 at all T, prose+IndentationError), KILL training immediately â€” do not wait for ckpt-200 to confirm.
- **[pattern:project]** Loss-line tail extraction from wandb-logged training: use `tail -40 $LOG | grep -oE '(ce_gap|kl_anchor|margin_loss).{0,18}'` â€” fixed-length context after each key works around the dict format. Avoid `grep -oE "'(ce_gap|...)':\s*[-0-9.e]+"` because the quote-escape inside `ssh "..."` outer command breaks the regex.
- **[correction:project]** Cross-vocab on-policy distill: forwarding the student on a freshly RIGHT-padded re-pack while sampling came from a LEFT-padded tensor silently corrupts training because RoPE puts the same logical token at a different absolute position. Always forward the student on the SAME left-padded `full_ids` used for sampling.
- **[env:project]** MoE allocator on DSC-V2-Lite leaves multi-GB reserved-but-unallocated fragments. Always export `PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True` in pod_runner shell scripts BEFORE activating the venv.
- **[gotcha:project]** Pod scripts importing the local `mythic_rdt` package need `PYTHONPATH=src` (or `sys.path.insert(0, "src")`) prepended. Bare `python -c "from mythic_rdt..."` from `/workspace/mythic-rdt` fails with `ModuleNotFoundError`.

- **[gotcha:project]** `torch.utils.checkpoint.checkpoint(..., use_reentrant=False)` truncates the backward graph if NO input tensor to the checkpointed function requires grad. With NF4-frozen base + frozen prelude, `h` arriving at the recurrence loop has `requires_grad=False` even though LoRA modules INSIDE `_loop_step` have trainable params â†’ backward fails / no gradient flows. Fix: `if use_ckpt and not h.requires_grad: h.requires_grad_(True)` immediately before the loop. HF Trainer sidesteps this via embed-input hooks; raw PyTorch loops must do it explicitly.
- **[gotcha:project]** `self.recurrence.to(dtype=base_dtype)` without `device=` leaves LTI's `A_diag`/`B`, LayerScale, and gate parameters on CPU after init â€” but the wrapper crashes at the FIRST iteration that actually exercises LTI (t=1, since `first_iter_identity` bypasses t=0). With curriculum starting at T=1 and only ramping to T>1 around step 15-30, the device crash hides for many steps. Always move recurrence cell with both: `self.recurrence.to(dtype=base_dtype, device=base_device)`.
- **[pattern:project]** Every joint/chain training script MUST run a tokenizer + base + wrapper + 1-microbatch-with-T>1 forward smoke test as the FIRST 30 s, BEFORE any precompute or curriculum-warmup compute. A v6V chain wasted 90 min of precompute because the downstream trainer failed in the first 30 s of training (tokenizer path bug); a second v6V launch wasted ~15 min reaching curriculum T>1 to discover the device-placement bug. Gate the chain on a `--smoke-test` flag that exercises T>1 explicitly.
- **[pattern:project]** Long pod-side rsync: launch with `nohup rsync ... > /tmp/log 2>&1 &` and watch via `until ! kill -0 $PID; do sleep 60; done` in a `Bash run_in_background:true` task â€” survives Claude turn boundaries. `--info=progress2` writes `\r`-overwritten lines, so for tailing use `tr '\r' '\n' | tail -3` to see the latest bar.
